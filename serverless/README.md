# Serverless

![Cloud](https://img.shields.io/badge/Cloud-AWS-FF9900?logo=amazonaws&logoColor=white)
![Compute](https://img.shields.io/badge/Compute-Lambda-orange)
![API](https://img.shields.io/badge/API-API%20Gateway-blue)
![Database](https://img.shields.io/badge/Database-DynamoDB-green)

Serverless enables applications to be built, deployed, and managed using managed AWS services without provisioning or maintaining servers.

AWS Lambda, API Gateway, DynamoDB, IAM, and CloudWatch work together to create scalable event-driven applications with secure API communication, monitoring, automated infrastructure management, reliable cloud integrations, and resilient application architectures.

The focus of this section is understanding how serverless architectures, APIs, automation, and event-driven applications work practically.

---

## Learning Objectives

By working through these demonstrations, I aim to:

- Understand serverless architectures
- Build REST APIs using API Gateway
- Deploy AWS Lambda functions
- Store application data in DynamoDB
- Configure IAM permissions
- Monitor applications using CloudWatch
